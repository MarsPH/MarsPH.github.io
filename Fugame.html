This team project features a modular inventory system with a hot bar, allowing players to drag and drop items while supporting keyboard controls. Items rely on ScriptableObjects for efficient management. Interfaces handle item interactions, which provides flexibility across different object types. Serialized data preserves items across scenes. Clue management also utilizes ScriptableObjects, which allows the clues to be stored in the inventory.

For menu interaction, inventory slots, and clues, I structured them as a single journal system in this project. This journal enhances the story's narrative and integrates seamlessly with UI animations tied to the Inventory System. The system utilizes ScriptableObjects for data persistence, Interfaces for modular item interactions, and Event-Driven Architecture to manage UI updates efficiently. Serialized Fields ensure data retention across game sessions, while Coroutines handle smooth UI transitions.
