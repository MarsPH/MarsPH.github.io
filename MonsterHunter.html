In my Monster Hunter project, I applied core object-oriented programming principles alongside C# features to create a structured and efficient game system. I used encapsulation with private fields and controlled access methods, while abstraction keeps the logic modular across Hunter, Monster, Map, and GameForm. Inheritance defines Character as a base class, allowing polymorphism where MoveCharacter() behaves differently for Hunter and Monster. Composition integrates equipment like sword, shield, and potion into the Hunter.

The game runs on a multithreaded system, that handles monster movement asynchronously with Threading (Thread, Thread.Sleep). Monsters exhibit simple AI behavior with randomized movement. A game loop continuously processes player input, while exception handling (try-catch) ensures smooth map loading. The game also uses lists (List<Monster>) to track entities dynamically, maintaining an optimized structure.
